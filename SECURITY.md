# Security Policy

## 🔒 Reporting Security Issues

If you discover a security vulnerability in this template, please **do not** create a public issue.

### How to Report

Please report security vulnerabilities by **opening a private issue**:

1. Go to [Issues](https://github.com/matheme-justyn/my-vibe-scaffolding/issues/new)
2. Use the **Bug Report** template
3. In the title, add `[SECURITY]` prefix
4. Provide detailed information about the vulnerability

**Example Title**: `[SECURITY] Potential XSS vulnerability in example code`

### What to Include

When reporting a security issue, please include:

- **Description**: Clear description of the vulnerability
- **Location**: Which file(s) and line(s) are affected
- **Impact**: Potential impact if exploited
- **Reproduction**: Steps to reproduce the issue
- **Suggested Fix**: If you have ideas on how to fix it

## 🛡️ Security Scope

This repository is a **template** for project scaffolding. Security considerations:

### In Scope
- Vulnerabilities in example code that could be copied to real projects
- Insecure patterns or practices demonstrated in the template
- Dependencies with known vulnerabilities (if any)
- Exposed sensitive information in example files

### Out of Scope
- Issues in projects created from this template (those are the responsibility of the project maintainer)
- General best practice suggestions (use Feature Request instead)

## ⚠️ Important Notes

### This is a Template
- This repository contains **example code and configuration**
- **You are responsible** for security in your own projects created from this template
- Always review and audit code before using in production

### No Sensitive Data
- This template should **never** contain real API keys, passwords, or credentials
- All example values are placeholders
- If you find actual credentials, please report immediately as `[SECURITY]`

## 🔄 Security Updates

Security-related updates will be:
- Released as **patch versions** (e.g., 1.0.1)
- Documented in [CHANGELOG.md](./.template/CHANGELOG.md)
- Tagged with `security` label in releases
